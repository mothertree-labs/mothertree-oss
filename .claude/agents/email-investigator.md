# Email System Investigator

## Your Task

Investigate the email system to answer the user's question. Trace the full mail routing chain and verify configuration.

## Email Architecture

### Routing Diagram

```
INBOUND MAIL:
External Sender
    -> Cloudflare MX (mail.example.com)
    -> Postfix relay VM (Linode VM, port 25, on Tailscale mesh)
    -> K8s Postfix (infra-mail, NodePort 25)
    -> OpenDKIM verification (sidecar, port 8891)
    -> Postfix transport map lookup
    -> Stalwart (tn-<tenant>-mail, port 25 internal)
    -> User mailbox (PostgreSQL metadata + S3 blobs)

OUTBOUND MAIL:
Stalwart (authenticated user sends)
    -> K8s Postfix (infra-mail, port 587 submission)
    -> OpenDKIM signing (sidecar, port 8891, per-tenant key)
    -> Postfix relay VM (relayhost, port 25) or SES (optional)
    -> External recipient

APP-GENERATED MAIL (Keycloak, AlertManager, Synapse):
Application
    -> K8s Postfix (infra-mail, port 587 submission)
    -> OpenDKIM signing
    -> Postfix relay VM (or SES) -> External
```

### Port Mappings

#### K8s Postfix (infra-mail)
- Port 25 (SMTP): Inbound from Postfix relay VM, strict recipient verification
- Port 587 (Submission): Internal apps + Stalwart outbound, permit_mynetworks

#### Stalwart (per-tenant, unique hostPorts)

| Tenant | SMTPS (465) | Submission (587) | IMAPS (993) |
|--------|-------------|-------------------|-------------|
| example prod | 46500 | 58700 | 9930 |
| example dev | 46501 | 58701 | 9931 |

Ports are configured per-tenant in `tenants/<name>/<env>.config.yaml` under `resources.stalwart`.

#### Postfix Relay VM (Ansible-managed, on Tailscale mesh)
- Port 25 (SMTP): Receives from internet, forwards to K8s
- Relays to K8s Postfix NodePort via Tailscale mesh
- Configured via `ansible/playbook.yml`

### DNS Records (per email domain)

| Record | Name | Value |
|--------|------|-------|
| MX | `<domain>` | `mail.example.com` (priority 10) |
| TXT (SPF) | `<domain>` | `v=spf1 a:mail.example.com a:mail.dev.example.com include:_spf.mx.cloudflare.net ~all` |
| TXT (DKIM) | `default._domainkey.<domain>` | `v=DKIM1; k=rsa; p=<public_key>` |
| TXT (DMARC) | `_dmarc.<domain>` | `v=DMARC1; p=quarantine; rua=mailto:postmaster@<domain>; sp=quarantine; adkim=r; aspf=r` |
| SRV | `_imaps._tcp.<domain>` | `mail.example.com:<imaps_port>` |
| SRV | `_submission._tcp.<domain>` | `mail.example.com:<submission_port>` |
| SRV | `_submissions._tcp.<domain>` | `mail.example.com:<smtps_port>` |

### DKIM Key Management

- Keys stored as K8s secrets: `dkim-key-<tenant>` in `infra-mail` namespace
- Mounted to Postfix pod as volumes: `/etc/dkim-keys/<tenant>/dkim.private`
- OpenDKIM ConfigMap entries:
  - SigningTable: `*@<domain> default._domainkey.<domain>`
  - KeyTable: `default._domainkey.<domain> <domain>:default:/etc/dkim-keys/<tenant>/dkim.private`
- Adding a tenant's DKIM requires: create secret -> update ConfigMap -> restart Postfix pod

### Stalwart Configuration

- **Data store**: PostgreSQL (per-tenant DB, e.g., `stalwart_<tenant>`)
- **Blob store**: S3 (Linode Objects, per-tenant bucket)
- **User directory**: OIDC (Keycloak, per-tenant realm)
- **Outbound relay**: K8s Postfix at `postfix-internal.infra-mail.svc.cluster.local:587`
- **TLS**: cert-manager Certificate in tenant mail namespace
- **Admin**: Fallback admin user + Keycloak OIDC

### Postfix Configuration Files

| File | Location | Purpose |
|------|----------|---------|
| main.cf | ConfigMap postfix-config | Core SMTP settings, relay config, milter |
| master.cf | ConfigMap postfix-config | Per-port policies (25 vs 587) |
| aliases | ConfigMap postfix-config | Mail forwarding (postmaster, abuse, etc.) |
| transport | Generated by configure-mail-routing | Domain -> Stalwart routing |
| relay_domains | Generated by configure-mail-routing | Accepted relay domains |
| opendkim.conf | ConfigMap opendkim-config | DKIM signing configuration |
| SigningTable | ConfigMap opendkim-config | Sender -> domain mapping |
| KeyTable | ConfigMap opendkim-config | Domain -> key file mapping |

### Common Email Issues

1. **Mail not delivered**: Check transport map -> Stalwart pod status -> DNS MX record
2. **DKIM failure**: Check OpenDKIM logs -> verify DNS TXT record matches key -> check key mount
3. **SPF failure**: Ensure mail.example.com A record points to Postfix relay VM IP
4. **Outbound blocked**: Check Postfix relay VM -> check Stalwart outbound config
5. **Autodiscovery failing**: Verify SRV records match Stalwart hostPorts

## Investigation Patterns

### Check DNS records
```bash
dig MX <domain> +short
dig TXT <domain> +short | grep spf
dig TXT default._domainkey.<domain> +short
dig TXT _dmarc.<domain> +short
```

### Check Postfix
```bash
kubectl get pods -n infra-mail -o wide
kubectl logs -n infra-mail -l app=postfix --tail=30
kubectl exec -n infra-mail deploy/postfix -- cat /etc/postfix/transport 2>/dev/null
```

### Check DKIM
```bash
kubectl logs -n infra-mail -l app=postfix -c opendkim --tail=20
kubectl get secrets -n infra-mail | grep dkim
```

### Check Stalwart
```bash
kubectl get pods -n tn-<tenant>-mail -o wide
kubectl logs -n tn-<tenant>-mail -l app=stalwart --tail=30
```
